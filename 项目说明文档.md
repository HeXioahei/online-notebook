## 一、项目基本信息

### 1.1 项目简介

本项目是一个基于 **Spring Boot 3.2 + Vue.js 3.3** 的**多级目录在线笔记本系统**，支持用户注册登录、创建无限层级的目录树结构、编写 Markdown 格式笔记并实时预览渲染等功能。系统采用前后端分离架构，适合学生整理课程笔记、程序员记录技术文档或任何需要系统化知识管理的场景。

### 1.2 运行方式说明

#### 前置环境要求
```bash
- JDK 17 或更高版本
- Node.js 16+ 或更高版本
- MySQL 8.0+
- Maven 3.6+
```

#### 本地运行步骤

**Step 1: 导入数据库（一键完成）**
```bash
mysql -u root -p < database/backup_with_create_db.sql
```

**Step 2: 配置后端连接**
编辑 `backend/src/main/resources/application.yml`，修改数据库用户名和密码：
```yaml
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/notebook_db?useSSL=false&serverTimezone=Asia/Shanghai
    username: root          # 你的数据库用户名
    password:              # 你的数据库密码
```

**Step 3: 启动后端服务**
```bash
cd backend
mvn clean install -DskipTests
mvn spring-boot:run
# 后端启动在 http://localhost:8081
```

**Step 4: 启动前端服务**
```bash
cd frontend
npm install
npm run dev
# 前端启动在 http://localhost:5173
```

**Step 5: 访问系统**
打开浏览器访问 `http://localhost:5173`，使用测试账号登录：
- 用户名：hjj / 密码：123456（含完整课程数据）

### 2.2 核心功能实现

#### 功能一：多级目录管理 ⭐核心亮点

**实现思路**：
使用 Vue 3 递归组件机制实现无限层级目录树，通过 Props 向下传递数据，通过 Events 向上传递用户交互。

**关键逻辑**：
```vue
<!-- DirectoryTree.vue -->
<template>
  <div v-for="item in items" :key="item.id">
    <!-- 当前层级渲染 -->
    <div :style="{ paddingLeft: 8 + 'px' }">
      {{ item.name }}
      <button @click.stop="$emit('toggle-directory', item)">📂</button>
    </div>
    
    <!-- 递归渲染子项 -->
    <DirectoryTree
      v-if="expandedDirectories.includes(item.id) && hasChildren(item)"
      :items="getChildrenItems(item)"
      :level="level + 1"
      @toggle-directory="$emit('toggle-directory', $event)"
    />
  </div>
</template>
```

**特点**：
- ✅ 支持任意深度的目录嵌套
- ✅ 等距缩进（每级8px），视觉层次清晰
- ✅ 展开/折叠交互流畅
- ✅ 同级目录可重名

#### 功能二：级联删除（递归算法）⭐核心亮点

**实现思路**：
采用深度优先搜索（DFS）的后序遍历策略，先删除子节点，再删除父节点，确保数据一致性。

**关键逻辑**：
```java
@Transactional
private void recursiveDeleteCategory(Long categoryId) {
    // 1. 递归删除所有子目录
    List<Category> childCategories = categoryRepository.findByParentId(categoryId);
    for (Category child : childCategories) {
        recursiveDeleteCategory(child.getId());
    }
    
    // 2. 删除当前目录下的所有笔记
    noteRepository.deleteByCategoryId(categoryId);
    
    // 3. 删除当前目录本身
    categoryRepository.deleteById(categoryId);
}
```

**特点**：
- ✅ 事务保护，要么全部成功，要么全部回滚
- ✅ 避免外键约束冲突
- ✅ 保证数据完整性

#### 功能三：Markdown 编辑与实时预览

**实现思路**：
使用 Marked 库解析 Markdown 文本为 HTML，通过 `v-html` 指令渲染到页面，自定义 CSS 确保图片自适应和代码块样式。

**特点**：
- ✅ 支持 GFM 标准
- ✅ 图片自适应显示
- ✅ 代码块语法高亮
- ✅ 实时预览流畅

#### 功能四：全局搜索功能

**实现思路**：
用户输入关键词后，前端发送 GET 请求到后端，后端在 `title` 和 `content` 字段进行模糊匹配，返回结果按更新时间降序排列，前端在弹窗中展示结果卡片并显示完整路径信息。

### 2.3 遇到的挑战与解决

#### 挑战一：多级目录的无限层级渲染

**问题描述**：
初始实现使用硬编码的三层 v-for 嵌套模板，导致第四级及以下的目录无法显示，且代码冗余难以维护。

**解决方案**：
学习并使用 Vue 3 递归组件机制，创建独立的 `DirectoryTree.vue` 组件，通过递归调用自身实现无限层级渲染。

**关键技术点**：
- 组件自身引用自身名称实现递归
- 通过 `level` prop 控制缩进距离
- 使用 `v-if` 条件渲染避免无限循环
- 所有事件通过 `$emit` 冒泡给父组件处理

**收获**：深入理解了 Vue 组件通信机制和树形数据的处理方式。

#### 挑战二：删除操作的级联处理

**问题描述**：
删除一个目录时，需要同时删除其下所有子目录、孙目录...以及所有关联的笔记，简单的 `DELETE FROM` 无法满足需求。

**解决方案**：
设计手动递归删除算法，采用深度优先搜索（DFS）策略，先删叶子节点（笔记），再删分支节点（目录），整个过程包裹在事务中保证一致性。

**关键技术点**：
- 后序遍历（先子节点后父节点）
- `@Transactional` 事务保护
- 避免外键约束冲突

**收获**：掌握了树形结构的遍历算法和数据库事务管理。
